title: 在 CapRover 上运行
---

[CapRover](https://caprover.com) 是用于 NodeJS、Python、PHP、ASP.NET、Ruby、MySQL、MongoDB、PostgreSQL、WordPress（等等）应用程序的极其易于使用的应用/数据库部署和 Web 服务器管理器！
它速度极快且非常强大，因为它在简单易用的界面背后使用 Docker、nginx、LetsEncrypt 和 NetData。

- 用于自动化和脚本的 CLI
- 易于访问和方便的 Web GUI
- 无锁定！移除 CapRover，您的应用继续工作！
- Docker Swarm 在幕后用于容器化和集群
- Nginx（完全可自定义模板）在幕后用于负载均衡
- Let's Encrypt 在幕后提供免费 SSL（HTTPS）

## 设置

CapRover Umami 设置相当容易。按照以下步骤在 CapRover 上设置 Umami：

### 1. 访问您的 CapRover 仪表盘

在 `captain.your-sub-domain.your-domain.xx` 登录您现有的 CapRover 实例账户。

如果您没有现有的 CapRover 实例，请查看 [CapRover - 入门](https://caprover.com/docs/get-started.html) 上的文档。

### 2. 从模板创建应用

点击 CapRover 实例的"应用"部分。
现在选择"一键应用/数据库"并搜索 Umami。您会发现 3 个选项：

- `umami_postgresql` 将创建带 PostgreSQL 数据库的 Umami。
- `umami_mysql` 将创建带 MySQL 数据库的 Umami。
- `umami_only` 将单独创建 Umami，您需要自行运行数据库基础设施并在设置中提供数据库凭据。

### 3. 使用数据库设置 Umami

从 CapRover 一键应用中选择 `umami_postgresql` 或 `umami_mysql` 后，系统会提示您进入设置页面，您可以在其中指定一些环境变量。注意其中大部分已经配置好了。

- `App Name` 是您在 CapRover 中的 Umami 实例的显示名称。
- `PostgreSQL 或 MySQL 版本`可以是 [PostgreSQL - Docker Hub](https://hub.docker.com/_/postgres) 或 [MYSQL - Docker Hub](https://hub.docker.com/_/mysql) 上可用的任何 PostgreSQL 或 MySQL 版本。默认值已成功测试可与 Umami 配合使用。
- `CapRover Umami 版本`指的是 [Umami 发布页面](https://github.com/umami-software/umami/releases)上找到的最新 CapRover Umami 版本。CapRover Umami 将使用为您的特定数据库发布的官方 Docker 镜像。与其使用版本号如 `v.1.33.x`，您可以使用 `latest` 来使用最新版本的 Umami 构建。
- `Database password` 提供您的数据库使用的密码。始终会有一个默认值供您使用。
- 可选：`PostgreSQL initdb 的参数`，您还可以提供 PostgreSQL initdb 的参数，例如 `--data-checksums`，此选项仅在 `umami_postgresql` 中可用。

现在只需点击"部署"即可开始设置带数据库服务的 CapRover Umami 实例。
请在完成之前不要离开页面。

### 4. 不使用数据库服务设置 Umami

在某些情况下，您可能需要自行管理数据库服务或为 umami 应用程序使用远程数据库服务，为此您可以使用 `umami_only` CapRover 应用。

在设置此应用程序之前，您需要使用 MySQL 或 PostgreSQL 创建数据库服务。

设置数据库后，在 CapRover 中点击 `umami_only` 并填写设置中的变量：

- `App Name` 是您在 CapRover 中的 Umami 实例的显示名称。
- `CapRover Umami 版本`指的是 [Umami 发布页面](https://github.com/umami-software/umami/releases)上找到的最新 CapRover Umami 版本。CapRover Umami 将使用为您的特定数据库发布的官方 docker 镜像。与其使用版本号如 `v.1.33.x`，您可以使用 `latest` 来使用最新版本的 Umami 构建。
- `Database Type` 根据您的数据库服务设置为 `postgresql` 或 `mysql`。
- `Database Remote URL` 是 Umami 可用于连接数据库的数据库服务远程 URL。如果您的数据库在您的 CapRover 网络中，请使用 `srv-captain--appName`。
- `Database User` 是您的数据库用户。
- `Database Table` 是为您 Umami 应用程序创建的表。
- `Database Password` 是您的数据库密码。

现在只需点击"部署"即可开始设置您的 CapRover Umami 实例。

请在完成之前不要离开页面。

### 4. 最终检查

您现在应该能够在 `app-name.your-sub-domain.your-domain.xx` 登录您的 Umami 部署，如[登录](/docs/login)部分所述。请立即在个人资料页面上更改您的密码。

### 5. 更新现有 umami 实例

在执行更新之前，首先从您的数据库进行备份。备份后，只需进入 CapRover 面板并选择您的 Umami 应用。
进入 `Deployment` 选项卡并向下滚动到 `Deploy via ImageName`，然后从 [Umami Docker Registry](https://github.com/umami-software/umami/pkgs/container/umami) 部署所需版本。

注意：不要忘记根据您的数据库选择正确的 Docker 镜像。通常 MySQL 镜像以 `mysql_` 为前缀，PostgreSQL 镜像以 `postgresql_` 为前缀。
