---
title: 在 Fly.io 上运行
---

[Fly.io](https://fly.io) 将容器转换为可以部署在一个或多个区域的虚拟机。`fly launch` 流程可以自动设置可与您的 Umami 应用连接的 Postgres 虚拟机。

## 使用 Umami 预构建容器进行设置

对于希望现在在 Fly 上部署 Umami 的任何人，这里有一个使用 [Umami 的 PostgreSQL 预构建容器](https://github.com/umami-software/umami/pkgs/container/umami/123905077?tag=postgresql-latest)的简单方法，而不是拉取源代码并修改 Dockerfile。

如果您还没有安装 `flyctl`，请[安装它](https://fly.io/docs/hands-on/install-flyctl/)。

1. 在您应用的新目录中，创建 `fly.toml`：

```toml
kill_signal = "SIGINT"
kill_timeout = "5s"

[experimental]
auto_rollback = true

[build]
image = "docker.umami.is/umami-software/umami:postgresql-latest"

[[services]]
protocol = "tcp"
internal_port = 3000
processes = ["app"]

[[services.ports]]
port = 80
handlers = ["http"]
force_https = true

[[services.ports]]
port = 443
handlers = ["tls", "http"]

[services.concurrency]
type = "connections"
hard_limit = 25
soft_limit = 20

[[services.tcp_checks]]
interval = "15s"
timeout = "2s"
grace_period = "1s"
```

2. `fly launch` 并选择使用找到的 `fly.toml` 配置创建应用。
3. 完成启动步骤，并在创建连接的 Postgres 应用的步骤上选择 `y`。
4. 暂时不部署应用。
5. `fly secrets set APP_SECRET="<任何字符串>"`，使用您想要的任何字符串来加盐哈希。
6. `fly deploy` 将会有错误，但让部署完成。
7. `fly scale memory 512`（或更高，如需要。Umami 在少于 512 MB RAM 时似乎会失败）
8. `fly deploy`
9. 按照 [Umami 文档](https://umami.is/docs/login)，使用用户：`admin` 和密码：`umami` 登录

根据 Fly [文档](https://fly.io/docs/)调整 Umami 应用的实例位置、机器数量、自动扩展、自定义域名以及任何其他 Fly.io 选项。
