---
title: 在 Vercel 上运行
---

[Vercel](https://vercel.com/) 是 Umami 使用的框架 [Next.js](https://nextjs.org/) 背后的公司。
他们还提供免费的托管服务，非常适合 Next.js 应用程序。

如果您还没有数据库，您可以创建 Vercel Postgres 数据库集成。您也可以按照[在 DigitalOcean 上运行](/docs/running-on-digitalocean)指南或
[在 PlanetScale 上运行](/docs/running-on-planetscale)指南来启动和运行数据库。
您也可以查看[托管](/docs/guides/hosting)下的**托管数据库**部分。

## 设置

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fumami-software%2Fumami&env=DATABASE_URL)

_使用上面的按钮自动执行步骤 1-5_

1. 将 [https://github.com/umami-software/umami](https://github.com/umami-software/umami) 项目分叉到您的 GitHub 账户。
2. 在 [Vercel](https://vercel.com/) 上创建一个账户。
3. 从仪表盘页面点击**导入项目**，然后指定您的项目在 GitHub 上的分叉 URL。
4. 将必需的环境变量 `DATABASE_URL` 添加到您的 Vercel 项目。这些值在[安装](/docs/install)的**配置 Umami**步骤中定义。您也可以在此处创建 Vercel Postgres 数据库。

- 您应该为 umami 使用 `POSTGRES_PRISMA_URL`，其形式为 `postgres://user:passwd@endpoint-pooler.postgres.vercel-storage.com/verceldb?pgbouncer=true&connect_timeout=10`。
- 数据库 URL 的环境变量可以在 `db/postgresql/prisma.schema` 文件中更改。

5. 部署并在 `<deploy-id>.vercel.app` 访问您的应用程序。
6. 按照**入门**指南从[登录](/docs/login)步骤开始，并务必更改默认密码。

### 通过 Vercel 代理 Umami Analytics

或者，您可以通过 Vercel 代理 Umami Analytics，从您的主域名为 Umami Analytics 服务。在使用 [Umami Cloud](https://umami.is/pricing) 时，这也有助于绕过广告拦截器。

请参阅 Vercel 的[重写文档](https://vercel.com/docs/project-configuration)。

在项目根目录中，添加包含以下内容的 `vercel.json` 文件：

```json
{
  "rewrites": [
    {
      "source": "/stats/:match*",
      "destination": "https://cloud.umami.is/:match*"
    }
  ]
}
```

追踪代码脚本标签应使用 `/stats/script.js` 作为 `src` 值：

```html
<script
  defer
  src="/stats/script.js"
  data-website-id="94db1cb1-74f4-4a40-ad6c-962362670409"
></script>
```
