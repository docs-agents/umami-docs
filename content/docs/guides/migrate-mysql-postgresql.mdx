---
title: 从 MySQL 迁移到 PostgreSQL
---

作为 Umami v3 的一部分，我们将移除 MySQL 作为支持的数据库选项，并标准化使用 PostgreSQL。
此更改确保所有环境中更一致和可扩展的数据层。

## 数据迁移

本文档概述了从 MySQL 到 PostgreSQL 的迁移过程，包括：

- 准备 MySQL 转储文件或 CSV。
- 验证数据完整性和应用程序兼容性。

目标是使过渡无缝进行，同时保留所有数据并确保最短停机时间。有开源工具如
`pgloader` 和 `pg_chameleon` 您可以探索。下面概述的过程是将数据纯粹迁移到新的 v2 PostgreSQL 数据库，
使此方法尽可能简单和独立。

## 要求

- MySQL 数据库架构必须与最新 v2 版本（v2.19.0）同步。
- 两个数据库的凭据：

```shell
MySQL: mysql://user:password@host:port/dbname

PostgreSQL: postgresql://user:password@host:port/dbname
```

## 使用 PostgreSQL 数据库安装 Umami

1. 在 PostgreSQL 服务器上安装 Umami（v2.19.0）。安装过程的文档可以在[此处](/docs/install)找到。
2. 截断以下表。这些表将从 MySQL 导入填充。

```sql
truncate table "_prisma_migrations";
truncate table "user";
```

## 运行迁移

### MySQL 转储文件

对于较小的数据库（少于 100 万条记录），您可以使用 MySQL 转储文件。官方文档可以在[此处](https://dev.mysql.com/doc/refman/8.4/en/mysqldump.html)找到。

1. 生成 MySQL 转储文件。

```shell
mysqldump --no-create-info --default-character-set=utf8mb4 --quick --skip-add-locks \
-u username -p mydb > mydbdump.sql
```

2. 将反引号替换为双引号以使其与 PostgreSQL 兼容。

```shell
sed -i 's/`/"/g' mydbdump.sql
```

3. 使用 psql 运行转储文件

```shell
psql -U username -d mydb < mydbdump.sql
```

Docker 版本

```shell
docker exec -i my-container psql -U username -d mydb < mydbdump.sql
```

现在您可以升级到 Umami（v3.0）

### 生成 CSV

对于较大的数据库，转储文件可能太大，插入语句可能变得低效。

1. 为所有必要的表生成 MySQL CSV 以及数据。

在 MySQL shell 中：

```sql
SELECT *
INTO OUTFILE '/tmp/website_event.csv'
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
FROM website_event;
```

**注意事项**

- 文件路径（/tmp/users.csv）位于 MySQL 服务器的文件系统上，不一定是您的主机。
- 确保 MySQL 有权限写入那里（secure_file_priv 可能限制此操作）。

2. 使用 `scp` 或 `cp` 将文件从 MySQL 服务器移动到 psql 可访问的位置。

3. 使用 psql `\COPY` 命令将 CSV 导入 PostgreSQL 数据库。为每个表重复命令。

```shell
psql -U postgres -d mydb -c "\COPY website_event FROM 'website_event.csv' CSV NULL '\N'"
```

**注意事项**

- 如果不使用默认架构，请包含架构。
