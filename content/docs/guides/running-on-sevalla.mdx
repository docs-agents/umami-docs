---
title: 在 Sevalla 上运行
---

[Sevalla](https://sevalla.com) 是一个面向开发者的 PaaS，提供[一键模板](https://docs.sevalla.com/templates/overview)、托管数据库和由 GKE 和 Cloudflare 提供支持的全球部署，无需处理复杂的基础设施。

Sevalla 支持 Git 基础部署、Dockerfile、容器镜像，开箱即支持大多数流行框架。本指南将引导您使用一键模板或在 Sevalla 上使用自己的 GitHub 仓库手动部署 Umami。

要开始，[创建一个 Sevalla 账户](https://sevalla.com)，您将收到 **$50 的免费使用额度**来试用。

## 方法 1：一键部署（推荐）

Sevalla 提供官方 Umami 模板，只需单击一下即可配置 Umami 应用和 PostgreSQL 数据库。

[![Deploy on Sevalla](https://sevalla.com//deploy-on-sevalla.svg)](https://app.sevalla.com/template/umami)

点击上面的按钮即可轻松开始。您的应用将在几分钟内上线。然后点击**访问应用**按钮并使用[默认凭据](https://umami.is/docs/login)登录。

## 方法 2：手动部署

如果您喜欢更多控制，可以从自己的 GitHub 仓库或公共 Umami 仓库手动部署。

### 设置 Sevalla 项目

1. 进入**应用程序** > **新建应用程序**
2. 选择 **GitHub 仓库**（如果您已分叉 [Umami](https://github.com/umami-software/umami)），或**公共仓库**并粘贴：`https://github.com/umami-software/umami`
3. 将分支设置为 `master`，输入应用名称，选择一个区域（与您的数据库相同），并根据需要选择 pod 大小。
4. 点击**创建**，但暂时跳过部署步骤，因为它会失败，因为我们还没有添加数据库。

### 数据库和部署

1. 进入**数据库** > **添加数据库**
2. 选择 **PostgreSQL**（推荐），选择与您的应用**相同的区域**，选择大小，然后点击**创建**。
3. 创建后，滚动到**已连接应用程序**，点击**添加连接**，选择您的 Umami 应用，并启用"添加环境变量"。
4. 将变量名设置为 `DATABASE_URL`（它可能预填充为 `DB_URL`，请更改为 `DATABASE_URL`）
5. 点击**添加连接**。这将使用正确的环境变量链接您的应用和数据库。
6. 返回您的应用程序并点击**部署**。构建完成后，访问您应用的 URL 并使用[默认凭据](https://umami.is/docs/login)登录。
