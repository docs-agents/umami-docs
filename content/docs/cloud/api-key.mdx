---
title: API 密钥
---

为了使用 Umami Cloud 访问您的数据，您需要生成一个 API 密钥。

参考部分中的所有 API 路由在 Umami Cloud 中都可用，使用 API 密钥，但 `Not Allowed` 部分中列出的一些路由除外。

## 创建您的密钥

登录后，点击侧边导航下拉菜单，然后点击**设置**。

![image](/images/cloud/navbar-settings.png)

进入 **API 密钥**并点击**创建密钥**按钮。

![image](/images/cloud/api-key-create.png)

您可以通过点击文本输入框中的`可见`图标来显示您的密钥。

![image](/images/cloud/api-key-details.png)

## 使用您的密钥

您可以通过查询以下地址来访问您的 Umami Cloud 数据。

`https://api.umami.is/v1`

只需通过自定义标头 `x-umami-api-key` 传入您的 API 密钥

例如，使用 `curl` 应如下所示：

```shell
curl https://api.umami.is/v1/websites
   -H "Accept: application/json"
   -H "x-umami-api-key: <api-key>"
```

## 将您的密钥与 API 客户端配合使用

设置以下必需变量以使 API 客户端使用您的 API 密钥。

```dotenv
UMAMI_API_KEY={Your API key}
UMAMI_API_CLIENT_ENDPOINT=https://api.umami.is/v1
```

有关使用 API 客户端的更多详情，请参阅 [API 客户端](/docs/api/api-client)。

## 限制

每个 API 密钥限制为每 15 秒 50 次调用。

## 不允许

以下 API 路由不可使用您的 API 密钥访问。

```text
/me/password
/users
/users/*
```

## 删除您的密钥

从 API 密钥中，点击要删除的特定密钥的`删除`按钮。

API 密钥一旦删除，将无法恢复。

![image](/images/cloud/api-key-delete.png)
