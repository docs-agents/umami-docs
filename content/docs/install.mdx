---
title: 安装
---

有几种不同的方式安装 Umami。

- **从源代码安装**：从 [GitHub](https://github.com/umami-software/umami) 获取代码并自行构建应用程序。
- **使用 Docker compose**：使用 `docker compose` 构建您自己的 Docker 容器。
- **使用 Docker 镜像**：下载预构建的 Docker 镜像。

## 从源代码安装

### 要求

- 运行 [Node.js](https://nodejs.org/) 18.18 或更高版本的服务器。
- 数据库。Umami 支持 [PostgreSQL](https://www.postgresql.org/)（最低 v12.14）数据库。

- **注意**：我们建议将 PostgreSQL 数据库配置为使用 **UTC** 时区。

    **为什么要用 UTC？**

    使用 UTC 可以避免区域偏移引起的问题，确保跨环境、服务和部署的时间戳一致且可预测。


### 安装 pnpm

```shell
npm install -g pnpm
```

### 获取源代码并安装依赖

```shell
git clone https://github.com/umami-software/umami.git
cd umami
pnpm install
```

### 配置 Umami

创建包含以下内容的 `.env` 文件

```dotenv
DATABASE_URL={连接 URL}
```

连接 URL 格式如下：

```dotenv
DATABASE_URL=postgresql://username:mypassword@localhost:5432/mydb
```

### 构建应用程序

```shell
pnpm build
```

首次运行构建时，它将在您的数据库中创建所有必需的数据库表。
它还会创建一个登录账户，用户名为 **admin**，密码为 **umami**。

### 启动应用程序

```shell
pnpm start
```

默认情况下，这将在 `http://localhost:3000` 启动应用程序。您需要从 Web 服务器
[代理](https://docs.nginx.com/nginx/admin-guide/web-server/reverse-proxy/) 请求，
或将[端口](https://nextjs.org/docs/api-reference/cli#production)更改为直接服务应用程序。

### 运行 Umami

您可以简单地运行 `pnpm start` 来启动 Umami，但强烈建议您使用进程管理器如 [PM2](https://pm2.keymetrics.io/)，
它将自动处理重启。

使用 PM2 运行：

```shell
pnpm --global add pm2
cd umami
pm2 start "pnpm start" --name umami
pm2 startup
pm2 save
```

## 使用 Docker 安装

Umami 附带了包含应用程序和 PostgreSQL 数据库的 docker compose。

要构建 Docker 容器并使用 Postgres 数据库启动，请运行：

```shell
docker-compose up -d
```

或者，如果您想使用预构建镜像，可以拉取带有 PostgreSQL 支持的 Umami Docker 镜像：

```shell
docker pull docker.umami.is/umami-software/umami:postgresql-latest
```
