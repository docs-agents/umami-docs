---
title: 身份验证
---

以下身份验证方法仅适用于自托管 Umami。对于 **Umami Cloud**，您只需
生成一个 [API 密钥](/docs/cloud/api-key)。

## POST /api/auth/login

首先，您需要获取一个 _token_ 以便发出 API 请求。您需要向 `/api/auth/login` 端点发出
包含以下数据的 `POST` 请求：

```json
{
  "username": "your-username",
  "password": "your-password"
}
```

如果成功，您应该会收到类似以下内容的响应：

```json
{
  "token": "eyTMjU2IiwiY...4Q0JDLUhWxnIjoiUE_A",
  "user": {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "username": "admin",
    "role": "admin",
    "createdAt": "2000-00-00T00:00:00.000Z",
    "isAdmin": true
  }
}
```

保存 token 值，并在所有数据请求中发送带有值 `Bearer <token>` 的 `Authorization` 标头。您的请求标头应如下所示：

```http request
Authorization: Bearer eyTMjU2IiwiY...4Q0JDLUhWxnIjoiUE_A
```

例如，使用 `curl` 应如下所示：

```shell
curl https://{yourserver}/api/websites
   -H "Accept: application/json"
   -H "Authorization: Bearer <token>"
```

每个需要权限的 API 调用都需要授权令牌。

---

## POST /api/auth/verify

您可以验证 token 是否仍然有效。

**示例响应**

```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "username": "admin",
  "role": "admin",
  "createdAt": "2000-00-00T00:00:00.000Z",
  "isAdmin": true,
  "teams": []
}
```
