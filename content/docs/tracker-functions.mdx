---
title: 追踪器函数
---

如果您想对追踪有更多控制权，Umami 追踪器暴露了一个可以在网站上调用的函数。默认情况下，所有内容都是自动收集的，但您可以使用 `data-auto-track="false"` 禁用此功能，并自己发送数据。
请参阅[追踪器配置](/docs/tracker-configuration)。

## 函数

```js
// 追踪当前页面
umami.track();

// 自定义 payload
umami.track(payload: object);

// 自定义事件
umami.track(event_name: string);

// 带数据的自定义事件
umami.track(event_name: string, data: object);

// 为当前会话分配 ID
umami.identify(unique_id: string);

// 会话数据
umami.identify(unique_id: string, data: object);

// 无 ID 的会话数据
umami.identify(data: object);
```

## 页面浏览

追踪一次页面浏览。

```js
umami.track();
```

默认情况下，追踪器自动收集以下属性：

- `hostname`：服务器主机名
- `language`：浏览器语言
- `referrer`：页面引荐来源
- `screen`：屏幕尺寸（例如 1920x1080）
- `title`：页面标题
- `url`：页面 URL
- `website`：网站 ID（必填）

如果您想发送自己的自定义 payload，请将对象传递给函数：

```js
umami.track({ website: 'e676c9b4-11e4-4ef1-a4d7-87001773e9f2', url: '/home', title: 'Home page' });
```

以上将仅发送属性 `website`、`url` 和 `title`。如果您想包含现有属性，请传递一个函数：

```js
umami.track(props => ({ ...props, url: '/home', title: 'Home page' }));
```

## 事件

追踪具有给定名称的事件。

```js
umami.track('signup-button');
```

## 事件数据

追踪带有动态数据的事件。

```js
umami.track('signup-button', { name: 'newsletter', id: 123 });
```

追踪事件时，默认属性包含在 payload 中。
这相当于运行：

```js
umami.track(props => ({
  ...props,
  name: 'signup-button',
  data: {
    name: 'newsletter',
    id: 123,
  },
}));
```

## 事件数据限制

事件数据可以处理任何 JSON 数据。为了保持性能，制定了一些规则。

- 数字最大精度为 4。
- 字符串最大长度为 500。
- 数组将转换为字符串，最大长度同样为 500。
- 对象最多 50 个属性。数组被视为 1 个属性。

## 会话

传入您自己的 ID 来识别用户。

```js
umami.identify('unique_id');
```

## 会话数据

保存有关当前会话的数据。

```js
umami.identify('unique_id', { name: 'Bob', email: 'bob@aol.com' });
```

要保存没有唯一 ID 的数据，只需传入 JSON 对象。

```js
umami.identify({ name: 'Bob', email: 'bob@aol.com' });
```
