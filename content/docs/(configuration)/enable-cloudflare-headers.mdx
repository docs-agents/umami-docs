---
title: 启用 Cloudflare 标头
---

如果您使用 Cloudflare 托管您的网站，Umami 将使用 Cloudflare 发送的
标头来确定访问者的位置信息。但是，默认情况下 Cloudflare 仅发送
国家/地区数据。如果您还想获取地区和城市信息，则需要
配置 Cloudflare 发送额外的标头。

## 步骤

1. 登录 Cloudflare 仪表盘并选择您的账户和网站。

2. 进入规则 > 设置。

3. 进入托管转换选项卡。

4. 启用**添加访问者位置标头**设置。

然后，您必须将[以下标头](https://github.com/umami-software/umami/blob/aaa1f9dc58feafe6af54248c5f0611112786fddf/src/lib/detect.ts#L14C2-L18C5)代理到 Umami：

- `CF-IPCountry`
- `CF-RegionCode`
- `CF-IPCity`

例如，如果您使用 Nginx 作为反向代理，可以添加以下配置：

```nginx
location / {
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Host $host;
    proxy_set_header CF-Connecting-IP $remote_addr;
    proxy_set_header CF-IPCountry $http_cf_ipcountry;
    proxy_set_header CF-RegionCode $http_cf_regioncode;
    proxy_set_header CF-IPCity $http_cf_ipcity;
    proxy_pass http://127.0.0.1:3000;
}
```

## 强制使用本地地理数据库

或者，当 Cloudflare 仅发送国家/地区但无法添加访问者位置
标头时，您可以将 `SKIP_LOCATION_HEADERS` 运行时变量设置为 `1`。
这将禁用标头检测并强制使用内部地理数据库。
