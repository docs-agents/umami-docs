---
title: 环境变量
---

您可以使用环境变量来配置 Umami。它们与您的 `DATABASE_URL` 放在同一个 `.env` 文件中。

---

## 运行时变量

运行时变量在 Umami 运行时被识别。您可以在启动应用程序之前设置环境变量。

### APP_SECRET

您安装的唯一值。

```
APP_SECRET = "random string"
```

### CLIENT_IP_HEADER

用于检查客户端 IP 地址的 HTTP 标头。当您在
使用非标准标头的代理后面时，这很有用。

```
CLIENT_IP_HEADER = "header name"
```

### COLLECT_API_ENDPOINT

允许您将指标发送到与默认 `/api/send` 不同的位置。这是为了帮助您避免一些[广告拦截器](/docs/bypass-ad-blockers)。

```
COLLECT_API_ENDPOINT = "/my-custom-route"
```

### CORS_MAX_AGE

CORS 预检请求应持续的秒数。默认为 24 小时。

```
CORS_MAX_AGE = 86400
```

### DATABASE_URL

```
DATABASE_URL = "connection string"
```

数据库的连接字符串。这是唯一的必需变量。

### DEBUG

应用程序特定区域的控制台日志记录。值包括 `umami:auth`、`umami:clickhouse`、`umami:kafka`、`umami:middleware` 和 `umami:prisma`。

```
DEBUG = "umami:*"
```

### DISABLE_BOT_CHECK

默认情况下，机器人被排除在统计之外。这将禁用机器人检查。

```
DISABLE_BOT_CHECK = 1
```

### DISABLE_LOGIN

禁用应用程序的登录页面。

```
DISABLE_LOGIN = 1
```

### DISABLE_UPDATES

禁用检查 Umami 的新版本。

```
DISABLE_UPDATES = 1
```

### DISABLE_TELEMETRY

Umami 收集完全匿名的遥测数据以帮助改进应用程序。如果不想参与，您可以选择禁用此功能。

```
DISABLE_TELEMETRY = 1
```

### ENABLE_TEST_CONSOLE

启用内部测试页面 `{host}/console`。需要管理员访问权限。用户可以手动向其网站触发页面浏览和事件。

```
ENABLE_TEST_CONSOLE = 1
```

### FAVICON_URL

用于显示网站图标的服务的 URL。

```
FAVICON_URL = "service URL"
```

默认为 `icons.duckduckgo.com`：

- https://icons.duckduckgo.com/ip3/{{domain}}.ico

您可以使用的一些替代方案：

- https://www.google.com/s2/favicons?domain={{domain}}
- https://logo.clearbit.com/{{domain}}

### GEO_DATABASE_URL

用于下载地理位置数据库的 URL。

### HOSTNAME / PORT

如果您在需要绑定到特定主机名或端口的环境中运行，例如 Heroku，您可以添加
这些变量并使用 `npm run start-env` 而不是 `npm start` 启动您的应用。

```
HOSTNAME = "my.hostname.com"
PORT = 3000
```

### IGNORE_IP

您可以提供逗号分隔的 IP 地址和范围列表，以从数据收集中排除。

```
IGNORE_IP = "IP addresses or ranges"
```

### LOG_QUERY

如果您在开发模式下运行，这会将数据库查询记录到控制台以进行调试。

```
LOG_QUERY = 1
```

### PRIVATE_MODE

禁用所有外部网络调用。注意，这也会禁用所有网站图标，因为它们来自 duckduckgo.com。

```
PRIVATE_MODE = 1
```

### REMOVE_TRAILING_SLASH

从所有传入的 URL 中删除尾随斜杠。

```
REMOVE_TRAILING_SLASH = 1
```

### TRACKER_SCRIPT_NAME

允许您为追踪脚本分配与默认 `script.js` 不同的自定义名称。这是为了帮助您避免一些[广告拦截器](/docs/bypass-ad-blockers)。

不需要 `.js` 扩展名。该值也可以是您选择的任何路径，例如 `/path/to/tracker`。

```
TRACKER_SCRIPT_NAME = "custom-script-name.js"
```

### SKIP_LOCATION_HEADERS

跳过使用已知位置标头进行国家/地区/城市检测，并强制使用本地地理数据库。

这在代理或 CDN 仅设置国家（没有地区或城市）标头的环境中很有用（如 Cloudflare 的 `CF-IPCountry`，当网络 > IP 地理位置切换为开时）。

```
SKIP_LOCATION_HEADERS = 1
```

## 构建时变量

构建时变量仅在构建过程中被识别。这包括构建自定义 Docker 镜像。您需要在构建应用程序之前设置环境变量。

### ALLOWED_FRAME_URLS

允许在 iframe 中托管应用程序的空格分隔的 URL 列表。

```
ALLOWED_FRAME_URLS = "URLs"
```

### BASE_PATH

如果您想在子目录下托管 Umami。您可能需要更新您的反向代理设置以正确处理 BASE_PATH 前缀。

```
BASE_PATH = "/custom"
```

### DATABASE_TYPE

```
DATABASE_TYPE = "postgresql"
```

要使用的数据库类型。这仅对 Docker 构建是必需的。

### FORCE_SSL

这将随所有请求发送 HTTP `Strict-Transport-Security` 响应标头。请参阅 [https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Strict-Transport-Security](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Strict-Transport-Security)。

```
FORCE_SSL = 1
```

### SKIP_DB_CHECK

跳过构建过程中的 `check-db` 步骤。用于 Docker 构建。

```
SKIP_DB_CHECK = 1
```

### SKIP_DB_MIGRATION

跳过构建过程中的 Prisma 迁移步骤。设置 `SKIP_DB_CHECK` 也会跳过此步骤。

```
SKIP_DB_MIGRATION = 1
```
